# راهنمای همگام‌سازی ابری — CorLink Accounting

این راهنما نحوه پشتیبان‌گیری و همگام‌سازی داده بین دستگاه‌ها را توضیح می‌دهد.

---

## ۱. داده‌ها کجا ذخیره می‌شوند؟

| محل | توضیح |
|-----|--------|
| **localStorage مرورگر** | محل اصلی ذخیره — کلید `financeDB` |
| **فقط روی همین دستگاه** | بدون sync، داده روی گوشی/لپ‌تاپ دیگر نیست |
| **پاک شدن** | با Clear Data مرورگر یا حذف cache ممکن است از بین برود |

> **توصیه:** هفتگی Export JSON یا CSV بگیرید.

---

## ۲. روش‌های همگام‌سازی

### روش A — کد همگام‌سازی (ساده، بدون حساب)

1. در **تنظیمات → همگام‌سازی** روی **کد همگام‌سازی** بکلیک کنید
2. کد Base64 در clipboard کپی می‌شود
3. در دستگاه دیگر، کد را در textarea بچسبانید و **بازیابی** بزنید

**مزایا:** رایگان، بدون اینترنت ثابت  
**معایب:** دستی — باید خودتان کد را منتقل کنید (واتساپ، ایمیل، …)

---

### روش B — JSONBin.io (ابری، خودکار)

سرویس رایگان [JSONBin.io](https://jsonbin.io) یک «سطل JSON» در ابر می‌دهد.

#### مرحله ۱: ساخت Bin

1. در [jsonbin.io](https://jsonbin.io) ثبت‌نام کنید
2. **Create Bin** → یک Bin خالی بسازید
3. **Bin ID** را کپی کنید (مثلاً `6789abcd...`)
4. از **API Keys** مقدار **X-Master-Key** را کopy کنید

#### مرحله ۲: تنظیم در اپ

1. **تنظیمات → همگام‌سازی ابری**
2. **شناسه JSONBin** = Bin ID
3. **کلید API** = X-Master-Key
4. **ارسال به ابر** — داده فعلی آپلود می‌شود

#### مرحله ۳: دستگاه دیگر

1. همان Bin ID و API Key را وارد کنید
2. **دریافت از ابر** — آخرین نسخه دانلود می‌شود

```
دستگاه A  ──push──►  JSONBin.io  ◄──pull──  دستگاه B
```

**نکته امنیتی:** Master Key را در اختیار کسی نگذارید. برای production از X-Access-Key با دسترسی محدود استفاده کنید.

---

### روش C — Export / Import فایل

| عمل | مسیر |
|-----|------|
| JSON | تنظیمات → خروجی JSON |
| CSV | تنظیمات → خروجی CSV |
| رمزنگاری‌شده | رمز عبور + Export → فایل `.json` |

---

## ۳. اعلان چک — چطور کار می‌کند؟

1. دکمه **فعال‌سازی اعلان چک** را بزنید
2. مرورگر مجوز Notification می‌خواهد → **Allow**
3. اپ هر **۱ ساعت** چک‌های با سررسید در **۵ روز آینده** را بررسی می‌کند
4. برای هر چک جدید، یک Notification نمایش داده می‌شود

### محدودیت‌ها

| مورد | توضیح |
|------|--------|
| مرورگر بسته | در iOS/Safari ممکن است اعلان نیاید |
| PWA نصب‌شده | روی Android معمولاً بهتر کار می‌کند |
| Background | Service Worker فعلاً push پس‌زمینه کامل ندارد |
| تکرار | هر چک فقط یک‌بار notify می‌شود |

---

## ۴. عیب‌یابی Sync

| خطا | راه‌حل |
|-----|--------|
| `Sync credentials not configured` | Bin ID و API Key را وارد کنید |
| `Push failed 401` | API Key اشتباه است |
| `Push failed 404` | Bin ID اشتباه یا Bin حذف شده |
| کد invalid | کد کامل کپی نشده — دوباره Copy کنید |

---

## ۵. سوالات متداول

**آیا داده روی سرور CorLink ذخیره می‌شود؟**  
خیر. فقط localStorage شما و (در صورت استفاده) JSONBin.io.

**آیا می‌توانم چند نفر همزمان sync کنند؟**  
JSONBin آخرین push را نگه می‌دارد — همزمانی real-time نیست. آخرین کسی که push کند برنده است.

**Backup رمزنگاری‌شده چیست؟**  
AES-256-GCM + PBKDF2 — بدون رمز عبور قابل بازگشایی نیست.
